### 目前优化的大体思路

1. 访问各个接口（除了登录以及注销接口）使用拦截器实现分辨用户是否未注销
2. 利用mybatis的xml方式实现动态查询
3. 对于前台对象dto以及查询对象do使用structmapper进行映射，减少代码量
4. 对各个实体类使用注解方式，减少代码入侵
5. 对service层使用事务来控制对多张表操作的一致性
6. 削减数据库表的数量，能合并尽量合并
7. 合并各个接口的返回实体对象dto，尽量使一个相关接口的返回保持一致
